Generating AppArmor profiles for Tor and Apache

vagrant up /staging$/
vagrant ssh app-staging
sudo su
cd /var/www/securedrop

(run tests, use the application, restart service, power off power on)

aa-logprof

(follow prompts and save at the end)

aa-complain /etc/apparmor.d/PROFILE_NAME

(run tests, use the application, restart service, power off power on)

aa-logprof

Repeat.

The AppArmor profiles are saved in /etc/apparmor.d/ There are two profiles: - usr.sbin.tor - usr.sbin.apache2

After running aa-logprof you will need to copy the modified profile back to your host machine.

cp /etc/apparmor.d/usr.sbin.apache2 /vagrant/install_files/ansible-base cp /etc/apparmor.d/usr.sbin.tor /vagrant/install_files/ansible-base

The AppArmor profiles are packaged with the securedrop-app-code. The securedrop-app-code postinst puts the AppArmor profiles in enforce mode. The app-test Ansible module (which is run as part of staging but not prod) puts the AppArmor profiles in enforce mode.
